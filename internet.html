Lesson 01

How the Internet Works

Understand what happens behind the scenes when you type a URL into your browser — from IP addresses to HTTP requests and web servers.

01

What is the Internet?

The Internet is a global computer network designed for storing, processing, and transmitting information. As of 2021, nearly 60% of the world's population are internet users — making web-related skills more in demand than ever.

Think of the Internet as a massive system of connected computers, constantly exchanging data using agreed-upon rules.

02

Clients and servers

Every connection on the Internet involves two types of computers:

🖥 Server

A dedicated computer always connected to the Internet. It stores files and content, and sends them to whoever asks.

💻 Client

Your computer, phone, or browser. It requests content from a server and displays it on your screen.

When you visit a website, your browser (the client) sends a request to a server. The server finds the file and sends it back. The file is not saved on your computer — it is displayed temporarily, so you always see the latest version.

03

IP addresses

Every computer connected to the Internet has a unique IP address — like a postal address for your device — so other computers can find it.

IPv4 — the most common format

Four numbers (0–255) separated by dots:

Example
91.198.174.192   ← IP address of Wikipedia

IPv6 — the newer format

IPv4 is running out (only ~4 billion addresses). IPv6 supports vastly more devices:

Example
1050:0:0:0:5:600:300c:326b   ← IPv6 address
Analogy: An IP address is like a phone number — every device has a unique one so it can be reached on the network.
04

Domain names

IP addresses are hard to remember. Domain names are human-friendly labels that map to an IP address — like a contact name replacing a long phone number. The DNS (Domain Name System) handles this translation automatically.

Example
# Both take you to the same website:
http://91.198.174.192    ← IP address (hard to remember)
http://wikipedia.org     ← Domain name (easy to remember)

In en.wikipedia.org:

  • .org — top-level domain (type of organisation)
  • wikipedia — the site's name
  • en — subdomain (English-language version)
05

Protocols

A protocol is a set of rules that defines how computers exchange data. Different types of data use different protocols.

ProtocolFull nameUsed for
HTTP / HTTPSHyperText Transfer ProtocolExchanging web pages
FTPFile Transfer ProtocolTransferring files
SMTPSimple Mail Transfer ProtocolSending emails
IMAPInternet Message Access ProtocolReceiving emails
IRC / IoTInternet Relay Chat / Internet of ThingsConnected devices
Analogy: Protocols are like languages. Two computers must speak the same protocol to exchange data — just like two people needing a shared language.
06

HTTP vs HTTPS

Both transfer web pages. The difference is security.

HTTP

Data is sent as plain text. Anyone intercepting the connection can read it. Not safe for passwords or payments.

HTTPS 🔒

Data is encrypted before sending. Even if intercepted, it cannot be read. Used by all modern websites.

Example
http://example.com   ← Not secure
https://example.com  ← Secure and encrypted ✓
Good news: GitHub Pages gives your site HTTPS automatically — for free.

Status codes

Every HTTP response includes a status code — a three-digit number telling the browser whether the request succeeded or failed.

CodeMeaningWhen you see it
200 OKSuccessEvery normal page load
301 MovedRedirectPage has permanently moved
404 Not FoundFile missingWrong URL or deleted page
500 Server ErrorServer crashedProblem on the server side
Tip: Create a file named 404.html in your repo to show a custom page when someone visits a URL that does not exist.
